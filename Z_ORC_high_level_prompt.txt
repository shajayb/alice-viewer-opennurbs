[GOAL]
Implement a C++ graphics application that authenticates with the Google Maps Platform, fetches, and renders Google Earth 3D Tiles for the city of London (specifically centering the camera on St. Paul's Cathedral).

[PLAN]
1. Security & Scaffolding: Configure `.gitignore` to protect credentials. Implement a secure runtime utility to parse `API_KEYS.txt` and extract the Google 3D Tiles API Key.
2. Network Integration: Integrate a minimal, cross-platform HTTP client (e.g., via standard libraries or lightweight headers) to request the root tileset JSON from the Google 3D Tiles API.
3. Tile Parsing & glTF Decoding: Implement parsing logic for the 3D Tiles bounding volume hierarchy and decode the initial spatial payload (extracting vertex, index, and normal data).
4. Render Pipeline Integration: Load the decoded geometry into the OpenGL data structures, ensuring strict zero-heap-allocation rules in the hot loop.
5. Camera Framing & Verification: Compute the bounding extents of the loaded St. Paul's Cathedral tile, explicitly set the camera target to frame it, and output the final `framebuffer.png` for visual verification.

[BOOTSTRAP]
Step 1: Create or update a `.gitignore` file in the repository root to explicitly ignore `API_KEYS.txt`. 
Step 2: Create a new header `include/ApiKeyReader.h`. Implement a robust, lightweight function that reads `API_KEYS.txt` from the root directory at runtime and extracts the Google API key.
Step 3: Modify `src/sketch.cpp` to invoke this reader. Print a masked version of the key (e.g., first 4 characters followed by asterisks) to the console to verify successful extraction without leaking the key in the logs.
Step 4: Ensure the core rendering loop is intact. Set up a default camera viewport and output an empty or default `framebuffer.png` to verify the build and execution pipeline is stable before we introduce HTTP dependencies. 
Step 5: Execute the local build via Ninja, ensure 0 compiler errors, and trigger the handoff.
