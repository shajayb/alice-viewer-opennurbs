# TERMINOLOGY DEFINITIONS
## SEMANTIC ANALYSIS
The process of evaluating the visual contents of a generated render. 
**Crucial Architectural Rule:** Semantic Analysis is NEVER performed by the C++ engine (e.g., via `libcurl` REST API calls) and is NEVER performed by auxiliary bloatware scripts (e.g., Python). 
Instead, Semantic Analysis is a manual, post-process action performed directly by the AI Agent (the Architect). Once the C++ executable finishes running and dumps the images, the Agent invokes its own native multi-modal capabilities to "look" at the output images.

## SEMANTIC IMAGE DESCRIPTION
The specific text generated by the AI Agent during Semantic Analysis that describes the visual contents of the image (e.g., "A close high-angle view focusing on the central dome of St Paul's Cathedral"). 
The C++ pipeline will initially output `output.json` with a placeholder (e.g., `"PENDING SEMANTIC ANALYSIS"`). It is the sole responsibility of the AI Agent to edit `output.json` post-execution and populate the `image_description` field with its generated description.

## COMPILE SUCCESS
The C++ application successfully compiles, executes, and writes the required structural outputs (`.bin`, `.3dm`, `.png`, and the baseline `output.json`) without crashing.

## SEMANTIC SUCCESS
A binary state of completion for a task. SEMANTIC SUCCESS is achieved only when:
1. **COMPILE SUCCESS** is achieved.
2. The AI Agent completes Semantic Analysis by viewing the images.
3. The AI Agent's resulting **SEMANTIC IMAGE DESCRIPTION** is sufficiently similar to the desired architectural contents requested by the user, proving the geometry and rendering logic is correct.
4. The AI Agent successfully injects this description into `output.json`.
