[GOAL]
Implement a C++ graphics application that authenticates with the Google Maps Platform, fetches, and renders Google Earth 3D Tiles for the city of London (specifically centering the camera on St. Paul's Cathedral).

[PLAN]
1. Security & Scaffolding: Configure `.gitignore` to protect credentials. Implement a secure runtime utility to parse `API_KEYS.txt` and extract the Google 3D Tiles API Key. (COMPLETED)
2. Network Integration: Integrate a minimal HTTP client to request the root tileset JSON from the Google 3D Tiles API. (COMPLETED)
3. Tile Parsing & glTF Decoding: Decode the spatial payload (extracting vertex, index, and normal data) and implement ECEF-to-ENU transforms to prevent floating-point jitter. (COMPLETED)
4. Hierarchical Traversal (BVH): Implement a memory-efficient tree structure to parse the `tileset.json` Bounding Volume Hierarchy. Evaluate geometric error/distance to selectively fetch high-resolution children tiles specifically intersecting London.
5. Render Pipeline Integration & Camera Framing: Render the traversed leaf nodes into the OpenGL pipeline. Compute the bounding extents of the St. Paul's Cathedral tiles, explicitly set the camera target to frame it, and output the final `framebuffer.png` for visual verification.


[BOOTSTRAP]
Step 1: (COMPLETED) Refactor `TilesetLoader.h` to define a `TileNode` structure capable of representing the Bounding Volume Hierarchy (BVH). It must store the tile's bounding volume (sphere/box), geometric error, transform, and content URI. Ensure these nodes are stored in a cache-friendly way (e.g., flat array/arena) to obey Data-Oriented Design rules.
Step 2: (COMPLETED) Implement a traversal function that evaluates a tile's bounding volume against a target coordinate. Convert the LLA coordinates of St. Paul's Cathedral (Lat: 51.5138, Lon: -0.0983) to ECEF, and write logic to recursively parse and fetch `children` tiles that contain or are closest to this target.
Step 3: (COMPLETED) Update the fetch logic to handle external tilesets. If a child tile's URI points to another `.json` file instead of a `.glb`, the loader must fetch and seamlessly graft that external tileset into the current BVH tree.
Step 4: Update `TilesetTest.h` to initiate this targeted traversal, accumulating all resolved `.glb` meshes into an array of `MeshPrimitive`s.
Step 5: Set the camera `focusPoint` to the ENU center of the St. Paul's tile cluster, loop through the active meshes in the `draw()` call, and capture `framebuffer.png`.
